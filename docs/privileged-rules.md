# 自定义特权规则 (Privileged Rules)

**核心处理文件**: `backend/recognition_engine/special_episode_handler.py`

特权规则是**优先级最高**的提取规则，在识别流程的最早期执行。命中后：
- **集数直接锁定**，不会被后续流程覆盖
- **标题作为优先搜索候选**，提高匹配准确率

---

## 🎯 规则格式

```
正则表达式 => {[字段=值;字段=值;...]}
```

使用 ` => {[...]}` 定义提取字段，支持捕获组引用（`\1`, `\2`）：

| 字段 | 说明 | 示例 |
| :--- | :--- | :--- |
| **group** | 字幕组名称 | `group=\1` |
| **title** | 标题 | `title=\2` |
| **e** | 集数 | `e=\3` |
| **s** | 季数 | `s=\1` |
| **tmdbid** | TMDB ID | `tmdbid=56559` |
| **type** | 媒体类型 | `type=tv` / `type=movie` / `type=auto` |
| **year** | 年份 | `year=2023` |

**type 字段说明：**
- `type=tv` - 强制使用 TV 接口搜索
- `type=movie` - 强制使用 Movie 接口搜索
- `type=auto` - 同时搜索 TV 和 Movie，根据 TMDB 返回结果自动判断类型

**示例：**
```
# 标准格式：[字幕组] 标题 - 集数
^\[([^\]]+)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}

# Yami.Shibai 格式：直接锁定 TMDB ID
Yami.Shibai.+?(\d+).+?(\d+).+?^[A-Za-z]+$ => {[tmdbid=56559;type=tv;s=\1;e=\2]}

# 电影格式：只提取标题
^\[([^\]]+)\]\s+(.+?)\s+\[BD\] => {[group=\1;title=\2]}

# 自动类型判断：同时搜索 TV 和 Movie
^\[([^\]]+)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3;type=auto]}
```

## 📝 常用规则模板

### 1. 标准格式 `[字幕组] 标题 - 集数`

**文件名示例：**
```
[LoliHouse] Goumon Baito-kun no Nichijou - 08 [WebRip 1080p].mkv
[MILKs&LoliHouse] Saioshi no Gikei - 12 [WebRip 1080p].mkv
```

**规则：**
```
^\[([^\]]+)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}
```

**解析：**
| 正则部分 | 说明 |
| :--- | :--- |
| `^\[([^\]]+)\]` | 匹配开头的 `[字幕组]`，捕获组 1 |
| `\s+` | 匹配空格 |
| `(.+?)` | 非贪婪匹配标题，捕获组 2 |
| `\s+-\s+` | 匹配 ` - ` |
| `(\d{1,4})` | 匹配 1-4 位集数，捕获组 3 |

---

### 2. 括号集数格式 `[字幕组] 标题 [集数]`

**文件名示例：**
```
[SweetSub] Mononoke - The Ashes of Rage [05][BDRip].mp4
```

**规则：**
```
^\[([^\]]+)\]\s+(.+?)\s+\[(\d{1,4})\] => {[group=\1;title=\2;e=\3]}
```

---

### 3. 指定字幕组定向规则

**文件名示例：**
```
[LoliHouse] Title - 01 [WebRip].mkv
```

**规则：**
```
^\[(LoliHouse)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}
```

---

### 4. 合作字幕组规则

**文件名示例：**
```
[NC-Raws & LoliHouse] Title - 01 [WebRip].mkv
```

**规则：**
```
^\[(.*?LoliHouse.*?)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}
```

---

### 5. 无字幕组格式 `标题 - 集数`

**文件名示例：**
```
Title Name Here - 08 [1080p].mkv
```

**规则：**
```
^([^\[\]]+?)\s+-\s+(\d{1,4}) => {[title=\1;e=\2]}
```

---

### 6. 带副标题格式 `[字幕组] 标题 [集数_副标题]`

**文件名示例：**
```
[晚街与灯] Title [05_大海啸].mkv
```

**规则：**
```
\[(晚街[與与]燈)\]\s*(.+?)\s*\[(\d{1,3})(?:\s*[-_][^\]]*)?\] => {[group=\1;title=\2;e=\3]}
```

---

### 7. 仅标题提取 (电影/剧场版)

适用于电影、剧场版等无集数的资源，只提取标题作为优先搜索词。

**文件名示例：**
```
[MyGroup] Movie Name [BDRip].mkv
[MovieGroup] Anime Movie [BD 1080p].mp4
```

**规则：**
```
^\[([^\]]+)\]\s+(.+?)\s+\[BD\] => {[group=\1;title=\2]}
```

**效果：**
- 标题作为优先搜索候选
- 集数由后续流程（Anitopy）处理
- 日志显示 `集数: 未锁定 (仅标题提取)`

---

### 8. 高级示例：直接锁定 TMDB ID 和类型

**文件名示例：**
```
Yami.Shibai.S01E02.Some.Title.1080p.WEBRip.x264.mkv
Yami.Shibai.S03E05.Another.Title.720p.WEBRip.AAC.mkv
```

**规则：**
```
Yami.Shibai.+?(\d+).+?(\d+).+?^[A-Za-z]+$ => {[tmdbid=56559;type=tv;s=\1;e=\2]}
```

**效果：**
- 直接锁定 TMDB ID 为 56559
- 媒体类型为 TV
- 季数和集数从捕获组提取
- 标题从后续流程自动识别

---

### 9. 多字段组合示例

**文件名示例：**
```
[MyGroup] Anime Title 2023 - 12 [1080p].mkv
```

**规则：**
```
^\[([^\]]+)\]\s+(.+?)\s+(\d{4})\s+-\s+(\d{1,4}) => {[group=\1;title=\2;year=\3;e=\4]}
```

**效果：**
- 提取字幕组、标题、年份、集数
- 所有字段都可用于后续匹配

## 🔧 正则编写指南

### 捕获组引用说明

正则中用 `()` 包围的部分是捕获组，索引从 1 开始：

```
^\[([^\]]+)\]\s+(.+?)\s+-\s+(\d{1,4})
   ↓        ↓          ↓
  组(1)    组(2)      组(3)
  字幕组    标题       集数
```

在规则中使用 `\1`, `\2`, `\3` 引用这些捕获组：

```
=> {[group=\1;title=\2;e=\3]}
```

### 常用正则模式

| 模式 | 说明 | 示例 |
| :--- | :--- | :--- |
| `\[` | 匹配 `[` (需转义) | `^\[` 匹配开头 `[` |
| `\]` | 匹配 `]` (需转义) | `[^\]]+` 匹配非 `]` 字符 |
| `\s+` | 匹配一个或多个空格 | `\s+-\s+` 匹配 ` - ` |
| `\d{1,4}` | 匹配 1-4 位数字 | 集数提取 |
| `(.+?)` | 非贪婪匹配任意字符 | 标题提取 |
| `[^\]]+` | 匹配非 `]` 的任意字符 | 字幕组提取 |
| `(?:\s|\[|$)` | 非捕获组，匹配空格/`[`/结尾 | 结束边界 |
| `(?:v\d)?` | 可选的版本号 | 匹配 `v2` 等 |

### 边界匹配技巧

| 需求 | 写法 |
| :--- | :--- |
| 匹配开头 | `^` |
| 匹配结尾 | `$` |
| 集数后是空格或 `[` | `(?:\s|\[|$)` |
| 可选版本号 `v2` | `(?:v\d)?` |

## ⚠️ 注意事项

### 1. 规则优先级

规则按**从上到下**的顺序匹配，第一条命中的规则生效。建议：
- 特定字幕组规则放在前面
- 通用规则放在后面

### 2. 标题验证

提取的标题必须满足：
- 长度 ≥ 2 个字符
- 不能为空

### 3. 集数锁定

特权提取的集数**不可被覆盖**，后续流程（Anitopy、合集增强等）都会跳过集数处理。

### 4. 多语言标题

如果标题包含 `/`（如 `中文标题 / English Title`），系统会自动拆分为多个搜索词依次搜索。

### 5. 捕获组引用

- 使用 `\1`, `\2` 引用正则捕获组
- 索引从 1 开始
- 确保引用的捕获组存在

## 🚀 使用方式

### 方式一：前端配置

在 **系统设置 → 识别与订阅规则 → 自定义特权规则** 中输入规则。

### 方式二：远程 URL

创建纯文本文件托管在服务器：

```
# 这是注释，会被忽略
^\[([^\]]+)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}

# LoliHouse 定向
^\[(LoliHouse)\]\s+(.+?)\s+-\s+(\d{1,4}) => {[group=\1;title=\2;e=\3]}

# Yami.Shibai 特殊格式
Yami.Shibai.+?(\d+).+?(\d+).+?^[A-Za-z]+$ => {[tmdbid=56559;type=tv;s=\1;e=\2]}
```

然后在 **远程 URL** 中填入文件地址，点击同步即可。
